# RULES.md

이 파일은 이 프로젝트에서 Claude와 협업할 때 반드시 따라야 하는 운영 규칙 문서다.

---

## 목적

- PRD, Design System, 코드가 항상 동기화된 상태를 유지한다.
- 문서 역할을 명확히 분리한다.
- 검증용 MVP 개발에 집중한다.
- 불필요한 설명으로 토큰을 낭비하지 않는다.

---

## 문서 역할 정의

### PRD.md

**포함 내용**

- 제품 목표
- 사용자 문제
- 기능 요구사항
- 사용자 플로우
- 정보 구조
- 정책
- MVP 범위
- 성공 지표

**포함하지 않는 내용**

- 색상
- 스페이싱
- 타이포그래피
- 컴포넌트 스타일
- UI 세부사항

---

### Design_System.md

**포함 내용**

- 디자인 원칙
- UI 규칙
- 컴포넌트 규칙
- 스페이싱
- 컬러
- 타이포그래피
- 인터랙션
- 모바일 UX 규칙

기능 요구사항은 포함하지 않는다.

---

### RULES.md

Claude 협업 규칙만 포함한다.

---

## 수정 요청 처리 규칙

사용자의 수정 요청이 들어오면 반드시 아래 순서를 따른다.

1. 변경사항 분류
2. 적절한 문서 업데이트
3. 코드 반영
4. 작업 완료 보고

변경사항 분류 과정은 사용자에게 설명하지 않는다.
단, 어떤 문서에 반영해야 하는지 애매한 경우에만 질문한다.

---

## 문서 반영 규칙

### 기능 변경 → PRD.md 반영

- 기능 추가
- 기능 제거
- 사용자 플로우 변경
- 정책 변경
- MVP 범위 변경

### 디자인 변경 → Design_System.md 반영

- 컬러 변경
- 스페이싱 변경
- 컴포넌트 스타일 변경
- 버튼 규칙 변경
- 카드 규칙 변경
- 모바일 UI 변경

### 협업 방식 변경 → RULES.md 반영

- 개발 프로세스 변경
- Git 규칙 변경
- 파일 구조 규칙 변경

---

## 절대 금지

- 디자인 세부사항을 PRD에 작성하지 않는다.
- 기능 요구사항을 Design_System에 작성하지 않는다.
- 동일한 내용을 여러 문서에 중복 작성하지 않는다.
- 코드만 수정하고 문서를 생략하지 않는다.
- 문서만 수정하고 코드를 생략하지 않는다.

---

## 개발 원칙

- 항상 PRD를 기준으로 개발한다.
- PRD에 없는 기능은 임의로 추가하지 않는다.
- 검증용 MVP 개발을 우선한다.
- 과도한 확장 설계를 하지 않는다.
- 완성도보다 사용자 검증 가능성을 우선한다.
- 최소 구현으로 최대 학습을 목표로 한다.

---

## UX 원칙

- Mobile First로 설계한다.
- 실제 모바일 앱 경험에 가깝게 설계한다.
- 디자인은 iOS 앱의 시각적 패턴을 기본 기준으로 한다.
- iOS에서 자연스럽게 보이는 UI를 우선 설계한다.
- Android 사용자도 불편 없이 사용할 수 있도록 구현한다.
- 플랫폼별 UI를 별도로 만들지 않는다.
- 단일 크로스 플랫폼 UX를 유지한다.
- 특정 플랫폼 가이드를 엄격하게 준수하기보다 일관된 사용자 경험을 우선한다.
- MVP 검증 목적이므로 모바일 웹(PWA 수준)을 기준으로 구현한다.
- 주요 화면은 390px 너비를 기준으로 설계하되, 360px~430px 범위의 모바일 기기에서 자연스럽게 동작해야 한다.
- 태블릿과 데스크톱은 보조 환경으로 지원한다.
- 사용자가 "앱처럼 느끼는 경험"을 제공하는 것을 목표로 한다.

---

## 코드 작성 원칙

- 기존 코드 스타일을 우선 따른다.
- 불필요한 라이브러리 추가를 최소화한다.
- 유지보수성을 고려한다.
- 컴포넌트를 적절히 분리한다.
- 하드코딩을 최소화한다.

---

## 응답 원칙

- 불필요한 설명을 최소화한다.
- 긴 분석을 하지 않는다.
- 구현을 우선한다.
- 사용자가 요청하지 않은 선택지 비교를 하지 않는다.
- 작업 완료 후 핵심 변경사항만 간단히 보고한다.
