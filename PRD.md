# Prototype PRD

## 프로젝트 개요

읽기의 필요성은 느끼지만 행동이 반복되지 않는 사용자를 대상으로,
즉시 읽기 → 반복 읽기 → 성장 보상 경험을 제공하여 반복 읽기 행동이 실제로 증가하는지 검증한다.

---

## 검증 가설

### H1. 시작 + 첫 반복

별도의 탐색 없이 콘텐츠에 즉시 진입할 수 있고,
읽기 행동이 부담 없이 완료 가능한 단위로 구성될 경우,
1회 읽기 이후 추가 읽기로 이어지는 비율이 증가할 것이다.

---

### H2. 연결 (반복 이어짐)

읽기 이후 별도의 탐색 없이 다음 행동으로 이어질 수 있다면,
세션 내 2회 이상 읽기 행동이 발생하는 비율이 증가할 것이다.

---

### H3. 동기 (반복 유지)

읽기 행동의 결과가 즉각적인 보상과 성장 변화로 제공될 경우,
보상 이후 추가 읽기 행동으로 이어지는 비율이 증가할 것이다.

---

## 오늘의 미션

### 설계 원칙

- 순차 진행 방식
- 이전 미션 완료 시 다음 미션 활성화
- 잠금 상태 미션 노출
- 첫 번째 미션은 최초 진입 시 기본 활성화
- 홈 화면 진입 시 현재 진행 중인 미션이 카루셀의 기본 노출 위치

### 미션 목록

1. 오늘 첫 읽기 완료하기
2. 오늘 한 번 더 읽기
3. 홈 추천 글 1개 읽기
4. 다음 글 이어 읽기
5. 퀴즈 1개 참여하기

### 상태

#### 진행 중

현재 수행 가능한 미션

- 진행 중 배지 노출

#### 완료

- 완료 배지 노출
- XP 지급 완료

#### 잠금

- 잠금 아이콘 노출
- 수행 불가 상태

### 상태 변경 규칙

**초기 상태**
- 미션 1 : 진행 중
- 미션 2~5 : 잠금

**미션 1 완료**
- 미션 1 : 완료
- 미션 2 : 진행 중

**미션 2 완료**
- 미션 1 : 완료
- 미션 2 : 완료
- 미션 3 : 진행 중

동일 패턴 반복

**순차 진행 원칙**
- 잠금 상태의 미션은 조건이 충족되더라도 완료 처리되지 않는다.
- 한 번의 행동에서 완료 가능한 미션은 현재 활성화된 미션 1개뿐이다.
- 현재 활성 미션의 완료 조건이 충족된 경우에만 해당 미션을 완료 처리한다.

### 미션 완료 처리

미션 완료 시

1. 해당 미션 XP 지급
2. 완료 상태로 변경
3. 다음 미션 활성화
4. 완료 상태 저장

### 미션 완료 조건

#### 미션 1 — 오늘 첫 읽기 완료하기

- 완료 조건 : `article_complete` 1회

#### 미션 2 — 오늘 한 번 더 읽기

- 완료 조건 : `article_complete` 2회

#### 미션 3 — 홈 추천 글 1개 읽기

- 완료 조건 : `home_recommendation` 영역에서 진입한 글의 `article_complete` 발생

#### 미션 4 — 다음 글 이어 읽기

- 완료 조건 : `next_article` CTA를 통해 진입한 글의 `article_complete` 발생

**다음 글 선택 원칙**
- 오늘 읽지 않은 글을 우선으로 노출한다.
- 오늘 읽지 않은 글이 없는 경우, 다음 순서의 글을 순환 노출한다.

#### 미션 5 — 퀴즈 1개 참여하기

- 완료 조건 : `quiz_submit` 발생

## 추천 콘텐츠

- 섹션명: 추천 글
- 서브 설명: 짧은 읽기부터 시작해 하루 한 걸음 성장해보세요.
- 추천 글 3개 노출
- 카드형 UI 사용
- 오늘 이미 읽은 글에는 읽음 상태를 표시한다.
  - 표시 방식: 썸네일 중앙에 원형 배경 + 체크 아이콘
- 카드 우측 상단에 카테고리와 읽기 시간을 표시한다.
  - 형식: `카테고리 · n분` (예: 성장 · 3분)
  - 카테고리는 카드 본문 영역에서 제거한다.

## 퀴즈 구조

- 콘텐츠당 퀴즈 1개
- 객관식 3지선다

### 퀴즈 데이터 구조

각 콘텐츠는 1개의 퀴즈를 가진다.

```json
{
  "id": "quiz_01",
  "question": "질문",
  "options": [
    "선택지1",
    "선택지2",
    "선택지3"
  ],
  "correct_answer": 1
}
```

`correct_answer`는 정답 선택지의 index를 의미한다.

### 퀴즈 결과 판별

사용자가 선택한 선택지와 `correct_answer`를 비교한다.

- 일치 → 정답
- 불일치 → 오답

### 퀴즈 결과 화면

#### 정답

🎉 정답입니다!

#### 오답

😢 아쉬워요

정답 선택지는 강조 표시한다.
사용자가 선택한 오답 선택지도 유지한다.

#### CTA

결과 확인하기

## 레벨 구조

### 설계 원칙

첫 성장 경험은 빠르게 제공하고
이후에는 반복 행동이 필요하도록 설계한다.

### 레벨 기준

| 레벨 | XP |
|---|---|
| Lv1 아메바 | 0 XP |
| Lv2 꼬물 물고기 | 20 XP |
| Lv3 리틀 몽키 | 120 XP |

---

## 성장 단계

| 레벨 | 기준 |
|---|---|
| Lv1 아메바 | 처음 시작 |
| Lv2 꼬물 물고기 | 20 XP 달성 |
| Lv3 리틀 몽키 | 120 XP 달성 |
| Lv4 꼬마 원시인 | 300 XP 달성 |
| Lv5 아인슈타인 | 600 XP 달성 |

프로토타입에서는 Lv1~Lv3까지만 실제 성장 가능하도록 구현한다.
성장 단계 보기 화면에서는 Lv1~Lv5 전체 단계를 노출한다.

## XP 지급 규칙

동일 행동에 대한 XP는 1회만 지급한다.

**중복 지급 제외 예시**
- 새로고침
- 화면 재진입
- 팝업 재노출

단, 서로 다른 미션은 각각 별도 행동으로 간주한다.

## 콘텐츠 데이터

기사 및 퀴즈 데이터는 추후 별도 전달 예정이다.

현재 단계에서는 콘텐츠 및 퀴즈 데이터 구조와 화면 연결을 구현한다.
실제 기사 및 퀴즈 데이터는 추후 전달 후 교체 가능하도록 구성한다.
콘텐츠 및 퀴즈 구조는 변경하지 않고 데이터만 교체 가능하도록 구현한다.
